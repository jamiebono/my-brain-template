# Link Validation Standards

## Purpose

Validates that all Obsidian wiki-style links `[[filename]]` point to files that actually exist in the vault. Prevents broken links from being committed and ensures the knowledge graph remains connected and navigable.

## Link Patterns Checked

This hook validates:

- **Simple wiki-links:** `[[filename]]`
- **Wiki-links with display text:** `[[filename|Display Text]]`
- **Nested path links:** `[[folder/subfolder/filename]]`
- **Links with headers:** `[[filename#section]]`
- **Links with blocks:** `[[filename^block-id]]`

## Check Logic

For each modified `.md` file in the commit:

1. **Extract all wiki-links** - Find all `[[...]]` patterns
2. **Parse link target** - Extract filename/path before `|`, `#`, or `^`
3. **Resolve path** - Check if target file exists in vault
4. **Verify extension** - Add `.md` if not specified (Obsidian convention)
5. **Report broken links** - List any links that don't resolve

## Examples

### Passing Examples

```markdown
See the [[todo]] for today's tasks
Reference: [[Projects/Clients/Cambridge-Associates/client-overview]]
Meeting notes: [[2026-01-03-kickoff|Kickoff Meeting]]
Jump to section: [[roadmap#Q1-2026]]
```

**Assumes these files exist:**
- `todo.md`
- `Projects/Clients/Cambridge-Associates/client-overview.md`
- `2026-01-03-kickoff.md`
- `roadmap.md`

### Failing Examples

**File doesn't exist:**
```markdown
See [[nonexistent-file]] for details

FAIL: nonexistent-file.md not found in vault
```

**Wrong path:**
```markdown
Check [[Projects/old-client/notes]]

FAIL: Projects/old-client/notes.md not found
```

**Typo in filename:**
```markdown
Reference [[todo-ist]] for tasks

FAIL: todo-ist.md not found (did you mean "todo"?)
```

## Remediation

When this check fails:

### Option 1: Fix the link
1. Verify the target filename
2. Correct the path/spelling
3. Use exact filename without extension (Obsidian adds .md)

### Option 2: Create the missing file
1. Create the referenced file in the appropriate location
2. Use proper template if applicable
3. Add required frontmatter

### Option 3: Remove the link
If the reference is no longer needed:
1. Remove the wiki-link brackets
2. Either keep as plain text or delete entirely

## Link Resolution Rules

The validator follows Obsidian's link resolution:

1. **Shortest path first** - Looks for filename in entire vault
2. **Relative paths** - Can specify `folder/file` for specificity
3. **Extension optional** - `.md` is assumed if not specified
4. **Case sensitive** - `[[Todo]]` â‰  `[[todo]]` on Linux systems

## Known Limitations

This hook does NOT validate:

- **External links** - `[text](http://example.com)` (markdown links)
- **Embed links** - `![[image.png]]` (media embeds)
- **Dataview queries** - Dynamic links generated by plugins
- **Alias resolution** - Links using frontmatter aliases

These may be added in future hook versions.

## Performance Considerations

For large vaults (1000+ files):

- Hook caches file index per commit
- Only validates modified files
- Reports all broken links at once (not one-by-one)

## Exclusions

The following are excluded from link validation:

- `.obsidian/` - Obsidian configuration
- `templates/` - Template files (contain placeholder links)
- `Assets/` - Media files
- `Excalidraw/` - Diagram files
