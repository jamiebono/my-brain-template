# Contributing to My Brain Template

Thanks for your interest in contributing! This project thrives on community input.

## Ways to Contribute

### Share Your Setup

- Open an issue or discussion showing how you've customized the template
- Share useful AI prompts you've developed
- Describe workflows that work well for you

### Improve Documentation

- Fix typos or unclear instructions
- Add examples for common use cases
- Translate to other languages

### Add Features

- New templates for common document types
- Additional AI agent roles
- Integrations with other tools

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your changes (`git checkout -b feature/my-improvement`)
3. **Make your changes** following the conventions below
4. **Test** that your changes work in both VSCode/Foam and Obsidian
5. **Submit a pull request** with a clear description

## Conventions

### File Naming

- Use `lowercase-with-hyphens` for files
- Templates start with `tpl-`
- Dates use `YYYY-MM-DD` format

### Markdown Style

- Use ATX-style headers (`#`, `##`, etc.)
- Include frontmatter in templates
- Use wiki-style links `[[like-this]]` for internal references

### Commit Messages

- Use present tense ("Add feature" not "Added feature")
- Keep the first line under 50 characters
- Reference issues when applicable (`Fixes #123`)

## What We're Looking For

**High value contributions:**

- Templates for common professional scenarios
- AI agent roles for specific domains
- Documentation improvements
- Bug fixes

**Please discuss first:**

- Major structural changes
- New dependencies
- Changes to core file conventions

## Questions?

Open an issue or start a discussion. We're happy to help!
